# SciWork 概要设计文档 v0.1

## 1. 产品定位

SciWork 是面向科学发现闭环的桌面工作台，形态上类比 Codex Desktop，但对象从代码仓库、终端和测试系统，转换为科学问题、知识空间、自动化科学装置、科学仪器、实验数据和科学技能包。

SciWork 不是科研管理系统，也不是普通文献问答助手。它的目标是帮助科学家完成从问题提出、文献检索、文献综述、假设生成、实验方案设计、装置执行、实验数据分析到下一轮实验建议的连续闭环。

一句话定位：

> SciWork 是科学发现驾驶舱：通过对话式 Desktop 调动知识空间、行动引擎和技能引擎，推进真实科学实验与科学发现。

## 2. 极简架构原则

SciWork 第一阶段坚持极简主义：

1. 一个用户只属于一个科学发现空间。
2. 用户登录后自动进入所属科学发现空间，不做多空间切换。
3. 一个科学发现空间对应一套自动化科学装置和一个主要学科领域。
4. 一个科学发现空间包含知识空间、行动引擎和技能组合。
5. 用户在科学发现空间内创建多个项目。
6. 每个项目拥有自己的文献库、知识库、实验记录、实验数据和实验记忆。
7. Desktop 只做驾驶舱，不做复杂科研管理后台。
8. 技能按项目和任务动态加载，不把复杂功能常驻在主界面。
9. 物理实验默认需要人工确认。
10. 实验数据、分析结果、失败经验和结论必须回流知识空间。

## 3. 核心概念

### 3.1 SciWork Desktop

SciWork Desktop 是科学家的驾驶舱，负责交互而不是承载复杂科学逻辑。

它负责：

- 用户登录和进入所属科学发现空间。
- 创建、打开和切换项目。
- 对话式推进科学任务。
- 展示文献证据、实验计划、风险和审批点。
- 展示装置状态、实验结果和下一轮建议。
- 动态加载技能 UI，例如实验画布、谱图查看器、分子/材料结构查看器、装置队列面板。

Desktop 应保持极简，复杂能力通过技能引擎按需加载。

### 3.2 科学发现空间

科学发现空间是用户所属的科学场域。第一阶段一个用户只能属于一个科学发现空间。

首批科学发现空间包括：

- iBioFoundry 科学发现空间：面向合成生物。
- iChemicalFoundry 科学发现空间：面向 MOF 材料发现。
- Oasis 科学发现空间：面向药物发现。
- 复旦晶泰科学发现空间：面向自动化化学反应。

科学发现空间定义：

- 所属学科领域。
- 绑定的自动化科学装置。
- 绑定的科学仪器。
- 默认 SCP Profile。
- 默认实验本体或 LabOntology。
- 默认安全和执行策略。
- 默认技能组合和可选技能集合。
- 用户可创建的项目类型。

一句话：科学发现空间定义用户所在的科学实验世界。

### 3.3 知识空间

知识空间是围绕某项科学研究发现的知识组织体系。它不是单纯知识图谱，也不是单纯文献库，而是把某个科学发现过程相关的公共知识、私域文献、实验记录、实验数据、假设、证据和结论组织在一起。

知识空间和基模共同组成 SciWork 的认知中枢：

- 基模负责自然语言理解、推理、规划、解释、写作和工具调用。
- 知识空间负责结构化上下文、证据链、科学记忆、实验数据关联和可追溯研究过程。

知识空间支撑：

- 科学问题提出。
- 文献检索。
- 文献综述。
- 假设生成。
- 实验方案设计。
- 实验数据分析。
- 结果解释。
- 下一轮实验建议。

知识空间包含两层：

- 空间级知识：公共 SciGraph-SCP、领域知识、LabOntology、共享方法和共享知识。
- 项目级知识：项目私域文献库、项目知识库、实验记录、实验数据、ExperimentGraph、结论和失败经验。

一句话：知识空间定义这个科学研究知道什么、为什么知道、还不知道什么。

### 3.4 行动引擎

行动引擎是科学发现空间的物理实验行动能力，由自动化科学装置、科学仪器和 SCP Profile 组成。

行动引擎负责：

- 声明装置和仪器能力。
- 查询可执行实验类型。
- 校验样品、试剂、参数和安全边界。
- 将实验意图编译为可执行实验任务。
- dry-run 或 simulation。
- 提交实验队列。
- 查询执行状态。
- 暂停、终止和异常处理。
- 获取实验结果。
- 将数据回流到知识空间。

默认执行策略是 Queue With Approval，即可以生成和排队实验任务，但关键物理执行动作需要人类确认。在设定条件下，支持 Bounded Autonomous Loop。

一句话：行动引擎定义这个科学发现空间能做哪些真实实验。

### 3.5 技能引擎

技能引擎是 SciWork 的可扩展能力系统，负责管理和加载科学技能包。

技能类型包括：

- 通用科学技能：文献检索、文献综述、假设生成、实验设计、结果解释。
- 领域技能：合成生物、MOF 发现、药物发现、反应化学。
- 数据分析技能：谱图分析、结构分析、活性数据分析、实验数据质控。
- UI 技能：实验设计画布、谱图查看器、分子/材料查看器、实验队列面板。
- SCP 服务技能：例如 SciGraph-SCP 等外部科学上下文服务。
- 硬件技能：自动化装置和科学仪器相关的特权技能。

技能加载原则：

- 科学发现空间提供默认推荐技能组合。
- 项目启用自己的技能集合。
- 任务执行时按需加载具体技能。
- 硬件相关技能属于特权技能，需要更严格的权限、安全和审计。

一句话：技能引擎定义 SciWork 可扩展的科学能力。

## 4. 核心对象关系

第一阶段的基本对象关系如下：

```text
User
  1 -> 1 Scientific Discovery Space

Scientific Discovery Space
  1 -> 1 Scientific Device
  1 -> 1 Discipline / Domain
  1 -> 1 Knowledge Space
  1 -> 1 Action Engine
  1 -> 1 Skill Set
  1 -> N Projects

Project
  1 -> 1 Project Literature Library
  1 -> 1 Project Knowledge Base
  1 -> N Tasks
  1 -> N Experiment Runs
  N -> N Enabled Skills
```

更直观地说：

```text
用户
  -> 唯一科学发现空间
      -> 一套科学装置
      -> 一个学科领域
      -> 一个知识空间
      -> 一个行动引擎
      -> 一个技能组合
      -> 多个项目
          -> 项目文献库
          -> 项目知识库
          -> 实验记录
          -> 实验数据
          -> 实验记忆
```

## 5. 登录与项目流程

```text
1. 用户登录 SciWork Desktop。

2. 系统根据用户身份自动解析所属科学发现空间。

3. Desktop 自动进入该科学发现空间。

4. 系统加载该空间的：
   - 科学装置
   - 行动引擎
   - 知识空间
   - 默认技能组合
   - 安全和执行策略

5. 用户创建或打开项目。

6. 项目加载：
   - 项目文献库
   - 项目知识库
   - 实验记录
   - 实验数据
   - 已启用技能

7. 用户在项目内发起对话任务或实验任务。

8. 基模与知识空间协同推理，技能引擎按需加载技能。

9. 如需真实实验，行动引擎通过 SCP Profile 校验并提交任务。

10. 实验结果回流项目知识库和知识空间。
```

## 6. 科学发现闭环

SciWork 的标准闭环如下：

```text
科学家提出目标
  -> 基模理解任务
  -> 知识空间提供文献、证据、历史实验和上下文
  -> 技能引擎加载相关科学技能
  -> 生成假设和实验方案
  -> 行动引擎检查装置能力和执行边界
  -> Desktop 展示计划并请求确认
  -> 自动化装置执行实验
  -> 实验数据回流
  -> 知识空间更新证据、结论和不确定性
  -> 系统建议下一轮实验
```

这个闭环是 SciWork 区别于普通 AI 科研助手的关键。

## 7. 与 SCP 和 SciGraph-SCP 的关系

SciWork 直接采用浦江实验室的 SCP 作为科学上下文协议基础，尤其用于外部科学资源、工具、装置和服务的协议化连接。

SciGraph 直接采用浦江实验室发布的 SciGraph-SCP 服务，作为公共科学知识图谱和科学上下文服务。

在 SciWork 中：

- SciGraph-SCP 是公共科学知识源。
- 知识空间是项目和科学发现空间的私域知识组织体系。
- 行动引擎通过装置 SCP Profile 暴露可执行实验能力。
- 技能引擎可以加载 SCP 服务技能和硬件技能。

SciWork 不重复建设公共 SciGraph，而是在知识空间中组织和对齐公共 SciGraph-SCP、私域文献、实验记录、实验数据和实验结论。

## 8. 权限与安全原则

物理实验比代码执行风险更高，因此需要明确的执行等级：

```text
Suggest
  只建议，不生成可执行任务。

Draft Protocol
  生成实验协议草案，但不提交装置。

Queue With Approval
  可生成并排队实验任务，执行前需要人类确认。

Bounded Autonomous Loop
  在固定参数空间、实验类型、资源预算、轮数和安全边界内自动迭代。

Emergency Stop
  任何时候可暂停或终止实验队列。
```

第一阶段默认采用 Queue With Approval；Bounded Autonomous Loop 只在设定条件下启用。

## 10. 第一阶段范围

第一阶段以四个具体科学发现空间为核心：

1. iBioFoundry：合成生物。
2. iChemicalFoundry：MOF 材料发现。
3. Oasis 绿洲一号：药物发现。
4. 复旦晶泰自动化装置：化学反应。

第一阶段的目标不是一次性做完整平台，而是建立统一 SciWork 架构，让四个科学发现空间可以用同一套 Desktop、知识空间、行动引擎和技能引擎抽象组织起来。
