# SciWork Desktop 极简浙大风 AI 科学工作台设计

日期：2026-06-10

## 目标

把当前 SciWork Desktop 原型继续优化为一个极简、中文、具备浙大气质的 AI 科学家工作台。整体使用习惯继续贴近 Codex Desktop / Claude Desktop：左侧导航，中间对话，右侧上下文和进度。

本轮只调整桌面界面、信息结构和视觉表达，不改变核心演示链路：

私域文献库 -> SciGraph 文献分析 -> 研究总结报告 -> 实验方案生成 -> LabOntology 任务校验 -> 模拟执行 -> Experimental Graph 结果回流。

## 体验原则

- 只呈现第一轮桌面演示真正需要的控件。
- 保留三栏结构，避免变成浏览器网页或营销页。
- 可见 UI 使用中文，SciGraph、LabOntology、Experimental Graph 等专有名词保留英文。
- 左栏承担结构导航，明确展示“科学发现空间 / 项目 / 会话”的关系。
- 视觉上以深色浙大蓝左栏为骨架，求是红作为动作和状态点缀，中间和右侧使用浅色纸面质感。
- IP 科学家形象作为“AI 科学助手”的身份露出，不做大面积装饰。
- 用克制的机器人实验、知识图谱、实验状态元素体现科学自动化和现代 AI 感。

## 信息架构

用户登录后默认进入自己所属的科学发现空间。科学发现空间以科学装置或实验设施为依据。

左栏展示三级结构：

1. 科学发现空间 / 装置
   - 示例：`自动化反应发现空间`
   - 展示当前装置身份和简短状态。

2. 项目目录
   - 每个项目对应一个工作区目录。
   - 示例：`温和偶联反应优化`
   - 可展示简短路径提示，例如 `projects/coupling-optimization`。

3. 会话
   - 每个项目下可以启动多个任务会话。
   - 示例：`文献分析`、`方案设计`、`模拟执行`。
   - 当前会话高亮显示。

左栏只保留两个关键上下文动作：

- `新建项目`
- `新建会话`

私域文献库、技能包、知识图谱资源等作为次级入口保留，但需要压缩展示，不能抢占主层级。

## 布局

### 桌面壳

Electron 原生顶部菜单栏隐藏。窗口打开后直接进入工作台。

三栏保持不变：

- 左侧：深色浙大风空间导航。
- 中间：浅色会话工作区。
- 右侧：浅色任务进度和项目上下文。

### 左侧栏

左侧栏使用深浙大蓝背景和轻微纹理。内容包括：

- `SciWork` 标识。
- 当前科学发现空间和装置信息。
- 当前空间下的项目目录。
- 当前项目下的会话列表。
- 紧凑的资源入口。

不使用卡片套卡片。结构分隔主要依赖留白、细线、短标题和小色块。

### 中间会话区

中间只服务当前会话，减少装饰。

内容包括：

- 紧凑顶部状态：当前项目、当前会话、执行状态。
- 对话线程。
- 科学任务主线，明确保留 SciGraph、LabOntology、Experimental Graph 出现的位置。
- 底部输入框，保留斜杠技能加载、模型选择、工作区、语音和发送入口。

### 右侧上下文区

右侧只展示当前任务有用的信息：

- 当前阶段进度。
- 项目上下文和目录。
- 文献、报告、方案、模拟执行状态。
- 知识资产：SciGraph、LabOntology、Experimental Graph。

右侧要像安静的状态面板，不做复杂仪表盘。

## 视觉系统

### 配色

- 左栏主色：深浙大蓝。
- 强调色：求是红。
- 中间和右侧：暖白、浅灰、纸面色。
- AI / 图谱辅助色：克制的青色或玉绿色线条。
- 文本：高对比墨色和中性灰。

界面不能只剩一整片蓝色，需要通过求是红、纸面色、墨线和少量图谱色形成层次。

### IP 形象

使用已有科学家 IP 形象资产，放在两个位置：

- Agent 消息头像。
- 输入框或右侧状态附近的小型“科学助手”身份标识。

IP 形象要表达“AI 科学家协作伙伴”，但不压过主工作区。

### 科学元素

科学感来自这些细节：

- 阶段标签和知识资产标签。
- 轻量图谱线条纹理。
- 小型机器人实验状态提示。
- `assets/` 中可替换的背景和 IP 图片。

后续替换背景或 IP 资产时，不需要改组件逻辑。

## 组件改造范围

### `electron/main.ts`

- 隐藏原生菜单栏。
- 保留现有 Electron / Vite 加载流程。
- 窗口尺寸原则上不改，除非 UI 适配需要小幅调整。

### `src/components/Sidebar.tsx`

- 可见文本中文化。
- 按“科学发现空间 -> 项目目录 -> 会话”重组。
- 压缩非核心资源区。
- 强化当前项目目录和当前会话的关系。

### `src/components/AgentThread.tsx`

- 可见文本和演示叙事中文化。
- 保留核心工作流阶段。
- 更明确地使用 IP 头像。
- 保持线程干净、可读。

### `src/components/ContextPanel.tsx`

- 可见文本中文化。
- 围绕当前任务压缩上下文内容。
- 保留 SciGraph、LabOntology、Experimental Graph 三个知识资产。

### `src/components/Composer.tsx`

- 可见文本中文化。
- 保留斜杠技能加载体验。
- 保留工作区、模型、语音、发送等 Codex/Claude 类使用习惯。

### `src/App.css`

- 降低视觉噪声。
- 左侧深色浙大风，中间和右侧浅色工作面。
- 加入克制的中国风、现代 AI 和科学自动化元素。
- 确保中文文本在桌面和较小窗口下不溢出。

## 数据和行为

本轮不新增真实后端能力。现有演示数据和内存状态可以继续使用。

界面需要表达清楚产品模型：

- 用户自动进入所属科学发现空间。
- 用户在空间中创建或选择项目目录。
- 用户在项目中启动多个会话。
- 每个会话可以执行科学工作流。

## 错误处理

本轮不增加新的错误状态。

如果图片资产加载失败，界面仍需保持文本可读，并使用现有 CSS fallback 样式。

## 验证

实现后需要验证：

- 单元测试更新到中文标签，并保留斜杠技能加载行为。
- 现有 workflow/state 测试保持通过。
- TypeScript 构建通过。
- Vite 生产构建通过。
- Electron 构建通过。
- 本地 Electron 预览确认：
  - 顶部原生菜单消失；
  - UI 已中文化；
  - 科学发现空间 / 项目 / 会话关系更清楚；
  - 左侧为深色浙大风；
  - 中间和右侧为浅色调；
  - IP 科学家形象可见但不喧宾夺主。

## 非本轮范围

- 真实登录。
- 真实文件系统项目创建。
- 真实模型调用。
- 真实硬件控制。
- 新生成 IP 或背景图片。
- 改变科学工作流业务逻辑。
